# 도구 셋업 & 활용 노하우

실제로 쓰면서 정리한 방법입니다. **계정 정보만 빼고 셋업·활용법은 그대로** 담았어요.
(아래 도구는 `02-skills-and-mcp.md` 의 플러그인·MCP와 연결됩니다.)

---

## GitHub (코드·문서 버전 관리, 공유)

- **커밋 규칙** : 한 커밋 = 한 종류 변경. 이것저것 섞지 말 것. 메시지는 "무엇을 왜" 한 줄
- **비밀파일 절대 금지** : `.gitignore` 에 `.env`, `*.key`, `credentials.json`, `*.db` 넣기 (이 레포 `.gitignore` 그대로 복사해 쓰면 됨)
- **브랜치** : main에 바로 쓰지 말고 브랜치 따서 작업 → 검증 후 합치기
- **자동화** : 클로드코드에 "커밋하고 PR 올려줘" 하면 `commit-commands` 플러그인이 다 처리

---

## Slack (실시간 알림 자동화)

- **목적** : 코드 실패·자동발행 에러·지표 하락을 실시간으로 받기
- **셋업** : 슬랙 워크스페이스에서 **Incoming Webhook** 생성 → 나오는 URL을 스크립트나 클로드에게 주면 그 채널로 메시지 발송
- **핵심은 "뭘 알릴지"** : 성공 알림은 노이즈. **실패 / 전일대비 지표 하락**처럼 "손 써야 하는 신호"만 보낼 것
- 채널은 용도별 분리 (예: 에러 전용, 지표 전용)

---

## Vercel (사이트 무료 배포)

- GitHub 레포와 연결하면 push 할 때마다 **자동 배포**
- **꿀팁** : 소스 레포는 **비공개로 둔 채** 사이트만 공개로 띄울 수 있음 (코드는 안 보여주고 결과물만 공개)
- 빌드 실패하면 클로드에게 로그 주고 "왜 실패했어" 물어보기
- (참고) Railway 같은 데서 파이썬 빌드가 자꾸 실패하면, 파이썬 버전을 `.python-version` 파일로 **고정**하면 해결되는 경우가 많음

---

## context7 (코딩 정확도 ↑)

- 클로드가 라이브러리를 쓸 때 **최신 공식문서를 그때그때 끌어옴** → 옛날 정보로 헛코드 짜는 걸 방지
- 그냥 깔아두기만 하면 됨. 따로 부를 일 없이 코딩할 때 자동으로 도움

---

## Canva / Figma (디자인)

- **색은 팔레트를 정해두고 그것만** 쓰기 (아무 색이나 쓰면 촌스러움). 예: 메인색 1 + 보조색 1~2 + 검정·회색
- **같은 위계의 텍스트는 크기 통일**, 강조는 크기 말고 색으로만
- 클로드에게 "이 팔레트로 만들어줘" 하며 색코드를 주면 일관되게 작업해줌

---

## Notion / Airtable (문서·데이터 관리)

- 반복 데이터(콘텐츠 큐, 고객 리스트 등)는 **표로** 관리하면 클로드가 읽고 자동 처리 가능
- "이 노션 페이지 읽어서 정리해줘" 식으로 활용

---

## Supabase / Stripe (웹앱 만들 때)

- **Supabase** : 무료 DB + 로그인 + 파일저장소. 웹앱 백엔드를 빠르게 붙일 때
- **Stripe** : 결제 붙일 때
- 둘 다 "앱을 직접 만든다" 단계에서 필요. 처음엔 신경 안 써도 됨
