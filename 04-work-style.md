# 작업 방식 & 규칙 (Work Style)

클로드한테 일 시킬 때 적용하는 규칙 전부. 이걸 `CLAUDE.md` 에 녹여두면 매번 설명 안 해도 클로드가 알아서 지킵니다.

---

## 1. 일 시키는 법 (워크플로우)

- **Scan → Plan → Approve → Apply → Verify** : 모든 물리적 변경 전에 이 순서 고정
- **DRY-RUN first** : 승인 전엔 파일 이동·생성·삭제 금지. "이렇게 바꿀게요"를 문서로 먼저 깔고, 승인된 것만 적용
- **No delete, archive only** : 지우지 말고 `05-archive/` 로 이동. 되돌릴 수 있게
- **1 commit = 1 change** : 한 커밋엔 한 종류 변경만 (폴더 이동 / 경로 수정 / 새 기능을 각각 별도 커밋). 회귀 생겼을 때 어디서 들어왔는지 추적됨
- **문서 먼저 (docs-first)** : 코드 한 줄 전에 마크다운 스펙부터. 인터페이스·데이터 흐름·게이트를 먼저 못박음
- **멈추자 제안 금지** : 한 단계 끝나면 "여기서 정리할까요" 묻지 말고 다음 단계로 계속
- **갈림길은 질문** : 추측하지 말고 핵심 갈림길마다 물어보고, 단계마다 검증
- **외부 서비스 대신 직접 셋업** : 남한테 시키는 대신 클로드가 직접 실행까지
- **플레이북 먼저 읽기** : 반복 작업은 시작 전 플레이북 통독. 즉흥 처리·헛시도 금지
- **파일 언급 시 풀경로** : 파일·문서 말할 때 항상 윈도우 풀경로 함께
- **무거운 일은 위임** : 파일 50개+ 리서치·무거운 분석은 서브에이전트(Agent)에 위임해 컨텍스트 보호

## 2. 글쓰기 규칙

- **줄표(—) 절대 금지** : 모든 문서(포스팅·기획서·노션·계획서)에서 em dash 금지. `:` `·` `→` `()` 쉼표·마침표로 대체. AI가 쓴 티가 나는 1순위 신호
- **사리지 말 것 (기본=대담)** : 과한 caveat·면책·"신중히" 줄글 빼고 직설적으로. 특히 블로그 글
- **쉬운 말로** : 개발 명령어·전문용어 나열 금지. 일상어 1~2스텝으로
- **보고 말고 결과** : 현황 줄글·할 일 리스트 나열 금지. 판단·실행까지 끝내고 결과로 말할 것. 사람은 물리적 최소 동작만
- **포스팅 가이드 형식** : 섹션당 텍스트 2줄 이내(이미지가 내용을 운반), 끝에 해시태그 블록, 바로 복붙 가능하게

## 3. 디자인 규칙

- **기본 폰트 Noto Sans KR** : 모든 HTML `<head>` 에 Google Fonts 링크 포함. (대안: Pretendard, Nunito 둥근맛, Noto Serif KR 클래식)
- **카드 컴포넌트 금지** : 흰 박스 + 왼쪽 컬러 보더 스타일 절대 금지 (AI 기본값, 앱 대시보드처럼 촌스러움). 텍스트 중심 레이아웃·구분선·배경 존으로 대체
- **같은 위계 = 같은 글자크기** : 같은 레벨 텍스트는 크기 통일, 강조는 **색·굵기만**. 같은 급에서 px 다르게 주지 말 것 (촌스러움 1순위)
- **광고소재는 포스터 언어로** : 박스/위젯/칩 금지. 큰 타이포가 주인공, 여백, 취소선(부담) vs 굵은 색(해방), 풀블리드 사진, 카드마다 표면 변주(풀블리드/플랫/다크넘버/화이트 에디토리얼). 다크 배경엔 밝은 글씨(검은 글씨 금지)
- **팔레트 정해두고 그것만** : CSS 변수로 브랜드색 정의(`--bg --surface --border --text --accent`). 예시 팔레트: `#fdf9f5`(베이지 배경) `#a8d8a8`(세이지) `#7abb7a`(포레스트) `#efe9e3`(보더) `#555`(본문)
- **플레이스홀더 이미지 사이즈 명시** : 이미지 슬롯 만들 때 권장 px 함께 표기 (AI 이미지 생성에 필요)
- **부드러운 애니메이션** : slideUp(0.3s ease-out), pulse-soft(2s loop) 류. linear·딱딱한 snap 금지
- **모바일 세이프에어리어** : `env(safe-area-inset-*)` 로 노치·하단바 대응

## 4. 콘텐츠 / 마케팅 규칙

- **리뷰 닉네임 랜덤화** : "인플루언서" 계열 닉네임은 반드시 무작위 닉네임으로 교체
- **오타·철자변형 키워드 살림** : 키워드 정리 시 완전중복만 제거, 오타·변형은 다 살림 (검색량 잡힘)
- **전일대비 하락 = 액션 트리거** : 그로스 지표를 절대값 아닌 **델타(변화량)** 로 봄. 전일 대비 떨어지면 그게 손 쓸 신호
- **발송 스케줄 고정** : 뉴스레터·정기 콘텐츠는 요일·시간 고정 (안 오면 이상한 루틴화가 목표)
- **콘텐츠 : 세일 = 3 : 1** : 3번은 읽을거리·브랜드 무드, 1번만 홍보. 매번 할인만 알리면 브랜드 목소리 약화

## 5. 클로드 활용 / 기록 습관

- **디자인 스킬 자동 활용** : UI/프론트 작업이면 `design-taste-frontend` 자동, 고급은 `/high-end-visual-design`, 미니멀은 `/minimalist-ui`, 다크 대시보드는 `/industrial-brutalist-ui`, 기존 개선은 `/redesign-existing-projects`
- **코드 잘림 방지** : 전체 코드 필요하면 `/full-output-enforcement`
- **사이클 번호 포스트모템** : 수정 이력을 `[NN차] (날짜)` 로 누적 기록. 구조: 증상 → 원인 → 수리 → 검증(실제 로그/ID) → 재발방지. semver 대신 사이클 번호라야 "몇 차에 회귀가 들어왔는지" 추적됨
- **결과는 영문+한글 혼용**
