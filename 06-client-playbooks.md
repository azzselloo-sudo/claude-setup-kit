# 클라이언트 작업 플레이북

클라이언트 일을 어떤 구조·순서·검증으로 하는지. 고객 내용은 빼고 **방법**만. 도메인이 달라도 뼈대는 같아서 한 번 익히면 재사용됩니다.

---

## 1. 표준 폴더 구조 (모든 작업 공통)

```
NN-클라이언트명/
├─ 01-docs/            계약·브리프·커뮤니케이션
│  ├─ TASK_BRIEF.md    목표·입력자료·일정
│  ├─ STATUS.md        현황·차단사항·진행상태
│  ├─ 01-입력자료/     고객 제공 원본 (마스킹 후 보관)
│  ├─ 02-지식/         지식 소스 (FAQ/아티클/규칙 등)
│  ├─ 03-분석/         로그분석·데이터·인사이트
│  ├─ 04-맵핑/         산출물 연결·매핑 테이블
│  ├─ 05-플레이북/     단계별 실행 가이드
│  ├─ 06-확인요청/     고객 검수용 엑셀
│  └─ 07-테스트/       QA 질문셋·검증보고
├─ 02-src/             생성 스크립트·프롬프트
├─ 03-data/            01-raw / 02-analysis
└─ 04-output/          최종 산출물·배포본
```
모든 작업 타입이 같은 구조 → 온보딩·인수인계 1회만 배우면 됨.

---

## 2. 표준 작업 흐름 (4단계)

1. **Scan** : 입력자료 수령·분류·전사. 원본 보존, 민감정보(로그인·가격·개인정보) 마스킹 후 분석용 txt/json 생성
2. **Plan** : 배포방식 결정, 데이터 기반 구조 설계, TASK_BRIEF에 경위·결정 기록
3. **Scaffold** : 생성 스크립트 실행, 자료 버전관리(xlsx+md 쌍 동기화), 확인요청 엑셀 작성
4. **Deliver** : 고객 회신 반영 → 어드민 직접 업로드 → 테스트 통과 → 라이브 + 변경이력 기록

---

## 3. 반복 작업 플레이북

### 챗봇 지식베이스 셋팅 (예: 상담 자동화 봇)
입력(고객 상담 로그) → 산출물(FAQ + 아티클 + 자동응답 규칙 + 워크플로우)
1. 로그 분석 → 질문유형·정책값 도출 (파서로 대화 추출 → 토픽/빈도 테이블)
2. 택소노미(분류체계) 확정 (업종별로 로그에서 실도출)
3. FAQ 실데이터 마스터 작성 (로그 근거값만 게시, 미확정은 `[확인필요]` 태그)
4. 아티클을 토픽과 1:1 맵핑 (토픽 = 아티클 = 봇 카테고리)
5. 규칙 설계 (자동응답 + 에스컬레이션 + 페르소나/언어 강제)
6. 배포방식 결정 (단순=빠른배포 / 다채널·유형버튼 필요=워크플로우)
7. 고객 확인요청 엑셀 (확정 vs 미확정 다중 검증)
8. 테스트셋으로 회귀 QA
9. 어드민 업로드 (Playwright 자동화 or 수동)
10. 배포 후 실대화 → 규칙 재학습 루프

### 카드뉴스·광고소재 생성
브랜드 brief + 레퍼런스 30~50장 + 상품 JSON → HTML 슬라이드 + PNG/MP4
1. 브랜드 정의 (컬러락·금지표현·타겟)
2. 소재 유형 마스터 (유형 5~7개 × 변주 규칙)
3. 레퍼런스 수집·디코드 (업종/톤/요소 태깅 → 엔진에 적재)
4. 생성 프롬프트 템플릿
5. 대량 생성 → 갤러리 비교표 → 픽
6. 선택된 TOP 3만 수작업 보강
7. 포맷 변환 (HTML → PNG/MP4/임베드)

### 블로그·채널 자동화
기획 CSV → 블로그 HTML + 카드뉴스 + 숏폼
1. 스케줄/웹훅 트리거 (예: 9/12/15시)
2. 큐 관리 (Sheets: 상품·키워드·상태)
3. HTML 생성 (강제 섹션 구조: hook/why/근거/리뷰)
4. 이미지 매핑 (생성 프롬프트 → 저장폴더 자동수집 → 슬롯 교체)
5. 멀티포맷 변환 (Playwright 캡처 / ffmpeg+TTS+자막)
6. 가이드 자동생성 + 데이터 수집 루프

## 4. 문서 템플릿

### TASK_BRIEF
- 0. 현재상태 (최신 진행 1문단)
- 1. 한 줄 요약 (목표·범위·기한)
- 2. 핵심 결정사항 (배포방식/기술/제약 테이블)
- 3~4. 입력자료 현황
- 5~7. 진행계획 & 스텝
- 백로그 (미정·고객대기·기술부채)

팀원이 "지금 어디서 뭘 기다리는가"를 1초에 파악 → 협업 병렬화.

### 고객 확인요청 엑셀 (6열)
| 번호 | 확인필요 항목 | 로그·근거 | 제안값 | 고객회신 | 적용상태 |
|---|---|---|---|---|---|
근거·제안·회신·적용을 한 표에 추적 → 실종 항목 0, 데이터 불일치 방지.

## 5. 품질 관리 & 검증

### 테스트셋 3단계 (A/B/C)
- **A. 자동응답** : "질문 → 올바른 지식·규칙 연결, 정확한 정책값 응답"
- **B. 에스컬레이션** : "민감/불량 상황 → 상담사 연결 트리거"
- **C. 할루시네이션 방지** : "미확정 항목 → '상담 후 안내'로 빠짐 (지어내지 않음)"

### 데이터 일관성 스캔
- 존재하지 않는 항목 참조 검사 (related_id 가 실제로 있나)
- 금지표현 스캔 (브랜드 금지어가 산출물에 없나)

### 라이브 배포 전 체크리스트
- [ ] 공개상태 수 일치 (게시 X개 / 비게시 Y개)
- [ ] 제목 정리 (파일명 흔적 제거)
- [ ] 규칙 옛 워딩 0개 (grep 검증)
- [ ] 워크플로우 미연결 노드 0개
- [ ] 테스트 A/B/C + 회귀 QA 통과
- [ ] 고객 미리보기 3명 이상 확인
- [ ] 문서·변경이력 최신화
- [ ] 배포는 업무시간 (장애 대응 가능 시간대)

---

## 6. 자동화 원칙 (데이터 생성)

- **단일 source of truth** : 마스터(xlsx) 하나에서 스크립트가 md/html/업로드용 포맷을 **동시 생성**
- **역방향 금지** : 어드민에서 수정하지 말고, 소스 고쳐서 재생성
- **생성 ↔ 선택 분리** : 생성은 1인 PC, 선택은 팀이 갤러리로. 역할 분리로 병렬화
