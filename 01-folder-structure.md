# 폴더 구조 표준

**원칙 : 모든 폴더는 `NN-이름` (숫자 먼저).**
정렬도 자동으로 되고, 한눈에 순서가 보입니다.

---

## 기본 구조

```
01-apps/      앱·서비스 프로젝트
02-scripts/   자동화·배치·원오프 스크립트
03-data/      DB·데이터 자산 (JSONL 등)
04-docs/      규칙·계획·운영 문서
05-archive/   삭제 대신 격리하는 곳
06-sandbox/   실험·임시 파일 (검증 후 승격)
07-clients/   클라이언트 프로젝트
              └── NN-클라이언트명/
                  ├── 01-docs/      계약서·브리프·커뮤니케이션
                  └── 02-tasks/
                      └── NN-태스크명/
                          ├── 01-docs/
                          ├── 02-src/
                          ├── 03-data/
                          └── 04-output/
```

---

## 신규 파일을 어디에 둘지

- 재사용할 스크립트 → `02-scripts/`
- 문서 → `04-docs/`
- 실험·임시 파일 → `06-sandbox/` (쓸만하면 정식 폴더로 승격)
- 지우고 싶은 것 → 지우지 말고 `05-archive/` 로 이동

---

## 만드는 법

클로드코드 창에 이렇게 치세요 :

> **"이 구조대로 폴더 만들어줘"**

그리고 위의 폴더 블록을 붙여넣으면 끝.
필요 없는 폴더는 빼고, 본인 일에 맞는 것만 골라도 됩니다.
(예 : 클라이언트 일이 없으면 `07-clients/` 는 빼도 됨)
