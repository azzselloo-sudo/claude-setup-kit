# AI 빌드 방법론 (How I Build)

AI 솔루션(바이브코딩)을 실제로 빌드하는 방식. 핵심 한 줄:
**강의·이론 A-Z 없음. 아래 규율을 실제 의뢰에 태워서 "배포·검증까지" 끝낸 걸 무한 반복.**

---

## 핵심 7단계

### 1. 시작은 "스펙·계획서 + DRY-RUN"
키보드 잡자마자 코드 안 짠다. 먼저 마크다운 스펙을 쓴다.
- 코드 한 줄 전에 인터페이스부터 못박음 (예: `pull_final() -> [{text, structure_id}]`)
- 데이터 흐름과 **게이트**를 먼저 정의
- 파일 건드리기 전에 "이렇게 바꿀게요"를 문서로 깔고(DRY-RUN), 승인된 것만 적용 (Proposed → Approved → Applied 블록)
- 이유: 계획서 없이 바로 시키면 AI가 100% 엉뚱한 데로 간다

### 2. 파이프라인은 "이름 붙은 순차 stage"로 분해 (인라인 로직 금지)
서버의 MVC에 대응하는 게 AI 쪽에선 **pipeline + orchestrator 패턴**.
```python
load_candidates()     # stage 1: 적재
add_candidate()       # stage 2: 중복제거하며 추가
classify_structure()  # stage 3: 구조 분류
ingest_track1()       # orchestrator: 위 stage들을 체이닝
```
핵심은 **각 stage가 독립적으로 테스트 가능**하게 쪼개는 것 (mock client 주입).

### 3. 스키마(데이터 계약)를 코드보다 먼저 확정
`design_system.json`, `card_schema.py` 같은 곳에 필드 계약을 **단일 소스**로 먼저 박는다.
생성기와 검증기가 같은 계약을 공유 → 필드가 비면 바로 티가 난다.

### 4. 검증을 아키텍처에 "내장" (사후 점검 X)
`validate_card()` 같은 날조 검증기를 코드에 박는다.
- 일반론·느낌표 남발·약한 카피는 자동 반려, `max_retry=3` 으로 재생성
- 없는 숫자는 절대 지어내지 않고 `●●` 로 비운 뒤 `missing_data` 에 기재
- AI의 "잘 만들었습니다"를 안 믿고, 코드 레벨에서 나쁜 출력을 거른다

### 5. 디버깅은 "가설 먼저, 패치 나중" (원인부터 배제)
증상에 반창고 붙이지 말고 **"원인이 X가 아니다"부터** 적는다.
- 예: 발행 쏠림 → 증상=한 카테고리만 쏟아짐 → "분류기 아님" 명시 → 진짜 원인=큐가 제품별 뭉텅이 → 라운드로빈 재배열
- 예: 빌드 실패 → 코드 버그 아니고 버전 자동선택 어긋남 → 버전 핀 고정으로 재발 차단

### 6. 수정마다 포스트모템을 문서로 ("메모" 말고)
가이드 문서 맨 아래에 `[NN차] (날짜)` 로 누적.
각 항목: **증상 → 원인 → 수리 → 검증(실제 Run ID 로그) → 재발방지**.
semver(v0.54) 대신 `[53차]` 라야 회귀가 몇 차에 들어왔는지 즉시 추적.

### 7. "학습 루프"를 중시
사람 손 거친 좋은 최종본을 다시 엔진에 먹여 다음 자동 초안이 좋아지게 한다.
- 수집(완성본 자동 후보화) → 승급(레퍼런스 refs에 반영)
- 사람은 **게이트에서만** 개입 (디자이너 추가동작 0)

### (보너스) 멀티 담당자 협업은 "축 하나"로 통합
여러 사람이 같은 프로젝트를 각자 다른 언어로 써오면 영영 안 만난다.
**공통 좌표계(축) 하나**를 정하고(예: 사용자 여정 #0~#9), 나머지 산출물을 "이 여정 어느 지점에서 필요한 결정이냐"로 흩어 넣는다.
같은 칸에 모이면 빠진 단계·정할 순서·영역 충돌이 저절로 드러난다.

## 코딩 컨벤션 (실전)

- **config 중앙화** : 상수·경로·헬퍼 함수를 `config.py` 한 곳에 집중 (`PRODUCT_KEYWORDS`, 경로 상수, `load_product()` 등)
- **중간 결과 JSON 저장** : `01_raw.json → 02_analyzed.json → 03_brief.json` 선형 연쇄로 **부분 재실행** 가능하게
- **큐/상태 추적** : `queue_state.json` 에 `used`, `failed`, `last_upload` 기록 → 재시도 대상 명시
- **조기 반환(early return)** : 빈 값·짧은 입력은 처리 전에 `return` (무의미한 API 호출 스킵)
- **부분 실패 격리** : 루프 안 try-except로 한 항목 실패해도 나머지 계속, 결과 리스트에 누적
- **경로는 pathlib** : `Path().expanduser()` 로 윈도우 백슬래시 자동 처리
- **격리 venv** : 충돌 잘 나는 도구는 전용 venv 분리(`.venv-svg` = Python 3.11), `.python-version` 으로 버전 명시
- **stdout utf-8** : 윈도우 한글 깨짐 방지로 `sys.stdout.reconfigure(encoding='utf-8')`
- **선택적 import 방어** : `try: import vtracer except: sys.exit("설치: pip install ...")`

## 자동화 아키텍처 패턴

- **Flask 대시보드 + 큐** : CSV/Google Sheets 큐를 읽어 HTML/PNG/TXT 다채널 생성. Sheets는 5분 캐시 폴링 → 비개발자 파트너는 시트만 편집
- **n8n 스케줄 워크플로우** : 스케줄 트리거 → Sheets 읽기(상태='대기') → LLM 생성 → 파일 저장 → 슬랙 알림 → 상태 업데이트. **1회=1항목**으로 비용 통제
- **Playwright HTML→이미지** : 렌더된 HTML을 슬라이드별 PNG 캡처 → 업로드 → API 발행까지 무인. 웹 자동화엔 `fill`(입력), `execCommand`(리치에디터), 키보드(`ArrowDown`+`Enter`)로 드롭다운
- **Socket Mode 슬랙봇** : 공개 URL 없이 봇 상주. 재시작에도 안전, 무상태는 자기 메시지 속 링크로 복원
- **Apps Script 폼 퍼널** : 폼 제출(onFormSubmit) → 즉시 메일 + 큐 등록 → 매일 정시 트리거로 후속 메일. 배포는 `clasp push`

## 프롬프트 엔지니어링 패턴

- **강제 JSON 스키마** : 파인튜닝 대신 "이 JSON 스키마로만 반환, 필드: a, b, c" 로 제약. `json.loads` 로 검증
- **판단 원칙을 프롬프트로 못박기** : 의미 해석은 LLM 지식에 맡기되, "무엇을 먼저·세게 볼지, 단정 금지" 같은 규칙으로 통제 → 매번 다른데 일관된 글
- **룩업표(하드코딩) 금지, 규칙+LLM** : "A면 B" 표는 조합 폭발·맥락 무시·뻔함을 부른다. 규칙으로 제약된 LLM이 매번 살아있는 글을 낸다
- **사실(fact)을 박아 날조 차단** : 글감마다 사실 기록 필드를 두고 AI는 그것만 쓰게. 없으면 `●●` 로 비움
- **레퍼런스 매칭 스코어링** : 임베딩 없이 업종 필터 → 타입 필터 → 톤 점수(키워드 유사도) → 요소 가산점으로 최적 레퍼런스 1장 선택해 구조·톤 모방
- **"어느 자리에 어떤 톤으로 왜" 분석** : "100자 카피 만들어" 오픈형보다, 레퍼런스의 자리별 역할을 분석해 "이 headline 자리에 X 톤"으로 지시하면 정확
