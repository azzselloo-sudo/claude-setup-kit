# 콘텐츠 & 그로스 방법론

콘텐츠를 어떻게 만들고, 리드를 어떻게 잡는지. 공식·프레임워크만 담았어요 (수치·브랜드 기밀 제외).

---

## 1. 콘텐츠 제작 공식

### 터지는 공식 (build-in-public 글감)
**일상 훅 → 발견 → 그래서 만들었어 → 솔직 에피소드 → 친구 질문/DM** 5단 구조.
통념깨기 + 정직성 + 구체성이 공유율을 만든다.
이유: B2B 신뢰는 좋아요가 아니라 **공유·DM·별(star)** 에서 온다.

### 선물형 vs 조언형 이원화
- **선물형** (코드/자료 공개) : 비핵심 도구일 때만
- **조언형** (코드 비공개) : 핵심 엔진·외주 보호용. 같은 공식, 다른 산출물
- 비공개 코드로도 완성글이 나온다. **질문 자체가 엔진.**

### 보이스 학습 = 정답지 모사
규칙 나열이 아니라 **실제 발행글 원문 2개를 옆에 두고 DNA 추출** :
반말 혼합 + 감정 + 고유 단어 + 경험부터 툭 + 번호 리스트 + 친구에게 묻듯 + 자격증명 먼저.
말투는 학습되는 것. 가이드라인보다 정답지 모사가 일관성을 만든다.

### 글감 뱅크 + 슬랙 채굴
팀이 실제로 하는 일(슬랙 대화)을 점수로 판별 → 글 초안까지 자동 생성.
토픽별 **사실 기록(fact)** 을 AI에 박아 날조 차단. "뭘 쓸까" 병목이 사라진다.

### 고객 니즈에서 출력 섹션 역산
매핑표(하드코딩) 말고, **고객 질문 5개 → 리포트 블록 5개 1:1 매칭**.
각 섹션의 목표를 먼저 못박고 생성. 규칙으로 제약된 LLM이 매번 살아있는 글을 낸다.

## 2. 그로스 / 리드 퍼널

### 이중 경로 퍼널 (배우기 ↔ 맡기기)
- 폼에서 **의향 먼저** 물음 (배우기 / 맡기기 / 기타)
- 메일에서 고민유형별 처방 + 의향별 CTA (배우기 → 스터디, 맡기기 → 무료상담)
- 이유: "그냥 해결을 원하는" 리드가 학습 콘텐츠만 받으면 이탈. 맡기기 경로가 수주로 전환

### 처방서: 백과식 X, 고민유형별 O
폼에서 고민 물어놓고 백과 9챕터 주던 미스매치 제거.
고민유형 4개로 나눠 **그 사람 문제 1개만** 처방. 톤은 전문가 존댓말(신뢰 → 맡기기).

### 리드 신호 = 좋아요 아니라 공유·DM·리포스트·별
비용이 드는 반응(공유·리포스트·DM)이 진짜 성의 시그널.
좋아요는 무의미. 유용성 기반 도달이 B2B 신뢰를 만든다.

### 슬랙 채널 + 보류(💤) 상태 추적
팀 채널의 링크/메모를 매일 리마인드. ✅처리 · 💤보류 · open 3상태로 추적.
자동화 피드에 떠내려가는 "다시 볼 것"을 놓치지 않게. 주 1회 보류 모아보기.

## 3. 콘텐츠 운영 규칙

- **콘텐츠 : 세일 = 3 : 1** (읽을거리 3, 홍보 1)
- **발송 스케줄 고정** (요일·시간 일정하게, 루틴화)
- **전일대비 지표 하락 = 액션 신호** (절대값 말고 델타로 봄)

---

## 4. 서비스화 / 상품화 사고법

- **기술이 아니라 서비스가 상품** : 코드 자체는 상품이 아님. 검증 스토리 + 영업 콘텐츠 + 운영이 결합돼야 마진 있는 상품
- **템플릿화로 멀티테넌트** : 하드코딩을 `brand_config.json` 으로 추출 → 고객마다 설정만 갈아끼움
- **"어디까지 외주, 어디는 내부"** : 전략(특히 마케팅)은 외주 금지 (주도권 상실). 회계·세무·인사·UX는 외주 OK. 외주가 너무 잘하면 주도권이 밖으로 넘어간다
- **수익화 단계** : MVP 운영 → 체험단 소수 → 베타 → 정식. 검증 스토리(실데이터)가 곧 영업 자료
